# Release Dock — منصة استضافة وتنزيل التطبيقات

متجر تطبيقات مستقل بواجهة احترافية، مع لوحة تحكم كاملة للإدارة، وعدّاد تنزيلات حقيقي 100% مبني على قاعدة بيانات فعلية (وليس أرقامًا وهمية). جاهز للنشر المباشر على **Railway**.

## المكوّنات

- **الخادم الخلفي:** Node.js + Express
- **قاعدة البيانات:** SQLite (عبر `better-sqlite3`) — بسيطة، سريعة، ولا تحتاج خدمة منفصلة
- **رفع الملفات:** Multer (الأيقونة، صور العرض، ملف التطبيق نفسه)
- **مصادقة لوحة التحكم:** JWT مخزّن في كوكي HttpOnly + كلمات مرور مشفّرة بـ bcrypt
- **الواجهة:** HTML/CSS/JS عادي (بدون إطار عمل) — سريعة التحميل وسهلة التعديل، بهوية بصرية خاصة (Release Manifest)

## التشغيل محليًا

```bash
npm install
cp .env.example .env       # ثم عدّل القيم داخل .env
npm run seed:admin admin "كلمة-مرور-قوية-هنا"   # ينشئ أول حساب مدير
npm start
```

الموقع: `http://localhost:3000`
لوحة التحكم: `http://localhost:3000/admin`

## النشر على Railway

### 1. إنشاء المشروع
- ادفع هذا المجلد إلى مستودع GitHub خاص بك.
- في Railway: **New Project → Deploy from GitHub repo** واختر المستودع.
- سيتعرف Railway تلقائيًا على Node.js عبر Nixpacks وسيشغّل `npm start` (معرّف أيضًا في `railway.json` و`Procfile`).

### 2. إضافة تخزين دائم (Volume) — **مهم جدًا**
النظام يخزّن قاعدة البيانات وملفات الرفع على القرص. حاويات Railway تُعاد بناؤها عند كل نشر، لذا **بدون Volume ستُفقد بيانات التطبيقات وعدّادات التنزيل** عند كل تحديث.

- من صفحة الخدمة في Railway: **Settings → Volumes → New Volume**
- اربطه بمسار `/data`
- هذا المسار سيحتوي قاعدة البيانات ومجلد الرفع، بحسب متغيرات البيئة أدناه.

### 3. متغيرات البيئة (Variables)
أضف هذه المتغيرات في تبويب **Variables**:

| المتغير | القيمة المقترحة |
|---|---|
| `NODE_ENV` | `production` |
| `JWT_SECRET` | نص عشوائي طويل وفريد (لا تستخدم القيمة الافتراضية) |
| `DB_PATH` | `/data/store.db` |
| `UPLOAD_ROOT` | `/data/uploads` |

`PORT` يديره Railway تلقائيًا، لا حاجة لتعيينه يدويًا.

### 4. إنشاء أول حساب مدير
بعد أول نشر ناجح، افتح **Shell** الخاصة بالخدمة داخل Railway (أو استخدم Railway CLI: `railway run`) ونفّذ:

```bash
npm run seed:admin admin "كلمة-مرور-قوية-هنا"
```

بعدها ادخل إلى `https://YOUR-DOMAIN.up.railway.app/admin` وسجّل الدخول.

### 5. النشر
كل `git push` إلى الفرع الرئيسي سيؤدي لنشر تلقائي جديد. البيانات تبقى محفوظة لأنها على الـ Volume.

## كيف يعمل عدّاد التنزيلات الحقيقي

- يبدأ كل تطبيق من `0` عند إضافته.
- العدّاد **لا** يتغيّر أبدًا عند تحميل الصفحة أو فتح تفاصيل التطبيق — فقط عند ضغط زر "تنزيل" الذي يستدعي مسار تنزيل الملف الفعلي.
- كل عملية تنزيل تُسجَّل كسطر في جدول `downloads` (بصمة IP مجهولة الهوية + الوقت)، ثم يُحدَّث `download_count` في نفس المعاملة على قاعدة البيانات (transaction)، فلا يوجد احتمال لتضارب أو عدّ مزدوج.
- ضُبطت مهلة قصيرة (15 ثانية لكل IP لكل تطبيق) لمنع تضخيم العدد بسبب الضغط المتكرر أو الروبوتات، دون التأثير على تسجيل التنزيل نفسه.
- كل الأرقام المعروضة للمستخدمين والإدارة مصدرها استعلام مباشر من قاعدة البيانات — لا يوجد أي رقم يُعرض بشكل ثابت أو عشوائي.

## لوحة التحكم

من `/admin` يمكن للمدير:
- رفع تطبيق جديد (اسم، وصف، إصدار، سجل تحديثات، أيقونة، صور عرض، ملف التطبيق)
- تعديل أو حذف أي تطبيق، مع حذف ملفاته من القرص تلقائيًا
- نشر التطبيق أو إبقاؤه كمسودة غير ظاهرة للزوار
- متابعة إجمالي التطبيقات، إجمالي التنزيلات، والأكثر تنزيلًا
- تسجيل دخول/خروج آمن عبر جلسة محمية بكوكي HttpOnly

## هيكل المشروع

```
server/
  index.js            نقطة الدخول والخادم
  db.js               تعريف قاعدة البيانات والجداول
  middleware/
    auth.js           التحقق من جلسة المدير
    upload.js         إعداد رفع الملفات (Multer)
  routes/
    apps.js           واجهات API العامة + مسار التنزيل الحقيقي
    admin.js           واجهات API الخاصة بلوحة التحكم
  utils/
    createAdmin.js     أداة سطر أوامر لإنشاء/تحديث حساب مدير
public/
  index.html / admin.html
  css/ js/
  uploads/             ملفات مرفوعة (أيقونات، صور، حزم التطبيقات)
```

## ملاحظات أمان قبل الإطلاق الفعلي

- غيّر `JWT_SECRET` إلى قيمة عشوائية طويلة قبل النشر.
- استخدم كلمة مرور قوية عند إنشاء حساب المدير، وغيّرها دوريًا عبر `npm run seed:admin`.
- فعّل نطاقًا مخصصًا بشهادة HTTPS من إعدادات Railway (مفعّل تلقائيًا على نطاق `up.railway.app`).
- إن توسّع عدد الزوار كثيرًا، يمكن لاحقًا الانتقال من SQLite إلى PostgreSQL المُدار من Railway بأقل تعديل على طبقة `db.js`.
